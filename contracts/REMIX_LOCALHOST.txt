Deploying to E Canna from Remix — correct flow (READ THIS)
==========================================================

!!! REQUIRED: EVM VERSION = london !!!
--------------------------------------
E Canna Mainnet (4111) and Testnet (4112) use Clique + stock Geth.
Genesis stops at **London** (no shanghaiTime). Remix **default** / shanghai /
cancun / prague / osaka will fail gas estimate or deploy.

FIX (Remix): Left panel **Solidity Compiler** → **Advanced Configurations** →
**EVM Version** = **london** (NOT "default").
Then **Compile** again → **Deploy & Run** → Injected Provider (MetaMask).

Hardhat / Foundry: set evmVersion to "london" (contracts/hardhat.config.cjs already does).

Without matching EVM, even a tiny contract may fail — even when MetaMask / RPC look fine.


TRY THIS FIRST (about 1 minute)
--------------------------------
Repo file: contracts/contracts/remix/RemixSmokeTest.sol
- No imports — paste / open that file in Remix.
- Compiler 0.8.24+ → EVM **london** → Injected Provider - MetaMask → Deploy.
- If this works = RPC + chain + balance + EVM are OK. Then try your real contract.


CODE / CONFIG (repo)
--------------------
1) ecnachain/metamask-network.json — chainId 0x100f (4111); rpc https://rpc.ecnascan.com
2) contracts/hardhat.config.cjs — evmVersion "london", PETH_RPC_URL / chain 4111
3) docs/DEPLOY-CONTRACTS.md — short user + agent guide
4) contracts/scripts/check-rpc.cjs — cd contracts && npm run check:rpc → should be OK


YOUR STEPS (order)
------------------
A) Node reachable.  npm run check:rpc  → OK

B) MetaMask network
   - RPC:     https://rpc.ecnascan.com  (testnet: https://testnetrpc.ecnascan.com)
   - Chain ID: 4111 (testnet: 4112)
   - Symbol:  ECNA (testnet: tECNA)

C) Remix
   - EVM Version = london
   - Compile
   - Environment: Injected Provider - MetaMask
   - Deploy

D) Verify on explorer → Verify & Publish → EVM london + matching bytecode


WHY NOT LIKE ETHEREUM / BNB DEFAULT?
------------------------------------
Those networks enable Shanghai+. Stock Geth Clique cannot — listing partners
hit "clique does not support shanghai fork". We stay on London so public sync works.
