Single-address setup (jo address tum doge, wahi sab jagah):

- ecnachain/.env: ECNA_PRIMARY_ADDRESS = wahi address

- server/.env: same ECNA_PRIMARY_ADDRESS (treasury + explorer “miner” label)

- genesis.json: extraData signer + coinbase + alloc = wahi address (genesis change = fresh datadir)

- docker/miner-private.hex: usi address ki raw private key (hex, no 0x) — bina iske sealing nahi chalegi
  (repo mein default Hardhat dev key hai — genesis signer 0x3e90… hai to is file ko usi address ki key se replace karo, warna seal match nahi karega)

- Funds (native): block reward + fees MINER_ETHERBASE par (compose = ECNA_PRIMARY_ADDRESS). Apni tx ka amount hamesha tx ke “To” par.

- Naya address: genesis mein coinbase + alloc + extraData signer teeno update. Clique order: 32 byte vanity + 20 byte signer + 65 byte seal (zeros) — geth source (extraVanity + signers + extraSeal). Nahi to “unauthorized signer”. Phir `.\reset-chain.ps1` ya geth + keystore wipe.

- Problem: blocks / funds nahi — (1) miner-private.hex mein leading space mat rakho, sirf 64 hex. (2) Purana galat import: container band karke `data/validator1/keystore` folder delete karo + `data/validator1/geth` delete karo, phir `docker compose up` (fresh import + init).

- Sabse common: purani chain chal rahi hai (genesis.json update ke baad bhi). `ecnachain` folder se: `.\reset-chain.ps1` chalao — phir MetaMask mein RPC `https://rpc.ecnascan.com`, Chain ID `4111`, tumhara address `0x3e90…` — genesis alloc balance dikhega.
